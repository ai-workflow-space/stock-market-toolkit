## Summary
Adds a language selector to the navbar enabling users to switch between English and Traditional Chinese (zh-TW). All navbar navigation labels are fully translated. Other user-facing strings (login, register, dashboard, alerts, settings, toasts, error messages) remain in English — this is a scoped partial implementation; full i18n coverage is tracked separately.

- **Language selector** in the navbar (top-right, globe icon) toggles `en` ↔ `zh-TW`
- **5 navbar label keys** translated: Dashboard, Signals, Compare, Alerts, Settings
- **Language preference persisted** to `localStorage` and reflected in `<html lang="">`
- Tooltip text on the language toggle button is also i18n-keyed

## Changes
- `frontend/src/i18n/` — i18next setup + `en.json` / `zh-TW.json` locale files
- `frontend/src/components/layout/Navbar.tsx` — language toggle button with `useTranslation()`
- `frontend/src/main.tsx` — import i18n init
- `frontend/package.json` — added `i18next@^24.2.3` and `react-i18next@^15.4.1`

## Partial implementation note
Only the navbar labels and language-toggle tooltip are translated. The following are intentionally out of scope for this PR and should be addressed in a follow-up:
- Login / Register pages
- Dashboard, Signals, Compare, Alerts pages
- Settings page and notification forms
- Toast notifications and error messages
- All backend error/detail strings
